Keep in mind...

Go right if you can; go left if you must

  • CONST e has type t if e is a constant with type t
  • VAR e has type t if e is a variable with type t
  • REF &e has type t* if e has type t
  • DEREF *e has type t if e has type t*
  • ARRAY e1[e2] has type t if e1 has type t* or type t[] and e2 has type int
  • STRUCT e.ai has type ti if e has type struct{t1 a1;...; tm am;}
  • APP e(e1,..., em) has type t if the return type of e is t and e1,..., em are subtypes of the types of the parameters in the function definition in the environment
  • OP e1 ◻ e1 has type t if e1 and e2 have type t
